# 服务层设计

## 概述

服务层在 ADB 协议层之上，提供各类 ADB 操作的高级 API。分为两类：

1. **Host 级服务**：与 ADB Server 直接通信（设备管理、端口转发等）
2. **设备级服务**：通过 AdbSession 与目标设备通信（Shell、文件传输、截图等）

## 一、HostServices

```kotlin
class HostServices(private val transport: Transport) {
    
    // 获取 ADB Server 版本
    suspend fun getVersion(): Int
    
    // 列出已连接设备
    suspend fun getDevices(): List<AdbDevice>
    
    // 持续追踪设备变化（Flow 长连接）
    fun trackDevices(): Flow<List<AdbDevice>>
    
    // 选择目标设备（后续操作路由到该设备）
    suspend fun selectDevice(serial: String)
    
    // 端口转发
    suspend fun forward(localPort: Int, remotePort: Int)
    suspend fun reverse(remotePort: Int, localPort: Int)
    suspend fun listForwards(): List<ForwardEntry>
    
    // 终止 ADB Server
    suspend fun kill()
}
```

### 文本协议解析

Host Service 使用简单的文本协议（4字节 hex 长度前缀）：

```kotlin
// 发送请求
fun buildRequest(service: String): ByteArray {
    val len = service.length
    val hex = "%04x".format(len)
    return (hex + service).toByteArray()
}

// 读取响应
suspend fun readResponse(transport: Transport): ByteArray {
    val header = ByteArray(4)
    // 精确读取 4 字节
    if (header.contentEquals("OKAY".toByteArray())) {
        // 成功，继续读取流数据的后续包
    } else if (header.contentEquals("FAIL".toByteArray())) {
        // 读取错误信息长度和内容
    }
}
```

### trackDevices 实现

```
host:track-devices 是长连接：
1. 发送 "host:track-devices"
2. 收到 OKAY
3. 循环读取：
   - 每次设备状态变化时推送新的设备列表
   - 格式同 host:devices: "serial\tstate\nserial\tstate\n"
   - 读到 length=0 表示连接断开
```

## 二、ShellService

```kotlin
class ShellService(private val session: AdbSession) {
    
    // 执行单条命令，等待返回完整结果
    suspend fun exec(command: String): ShellResult
    
    // 执行命令，流式返回输出
    fun execStream(command: String): Flow<ShellData>
    
    // 打开交互式 Shell
    suspend fun interactive(): InteractiveShell
}

class InteractiveShell(private val stream: AdbStream) {
    val output: Flow<ShellData>       // stdout + stderr + exit
    suspend fun writeStdin(input: String)
    suspend fun resize(rows: Int, cols: Int)
    suspend fun close()
}
```

### exec 实现流程

```
1. openService("shell,raw:<command>")
   或 openService("shell,v2,raw:<command>") (Shell v2)

2. 读取流中的 WRTE 包

3. 如果支持 Shell v2:
   - ShellProtocol 解析每个 WRTE payload
   - 分离 stdout/stderr/exit
   
4. 如果仅支持 Shell v1:
   - 所有输出在 stdout
   - 无 stderr 分离
   - 无 exit code (通过额外命令获取)

5. 收集完成 → ShellResult(stdout, stderr, exitCode)
```

### PTY 属性设置

通过 shell 协议发送窗口大小变更：

```kotlin
suspend fun resize(rows: Int, cols: Int) {
    // Shell v2: 在 stdin 流中发送窗口变更转义序列
    val escape = "[8;${rows};${cols}t".toByteArray()
    stream.write(escape)
}
```

> 注意：`export COLUMNS=` 仅影响 bash 环境变量，不会改变 PTY 的实际终端尺寸。

## 三、SyncService

```kotlin
class SyncService(private val session: AdbSession) {
    // 推送文件到设备
    suspend fun push(localPath: Path, remotePath: String, mode: Int = 0644)
    
    // 从设备拉取文件
    suspend fun pull(remotePath: String, localPath: Path)
    
    // 列出目录内容
    suspend fun listDir(remotePath: String): List<SyncEntry>
    
    // 获取文件信息
    suspend fun stat(remotePath: String): SyncEntry
}
```

### 推送流程 (SEND)

```
1. openService("sync:")
2. 发送 SEND + 远程路径长度 + 路径 + 权限模式
3. while (有数据):
     发送 DATA + 数据块长度 + 数据块
4. 发送 DONE + 文件时间戳
5. 接收 OKAY/FAIL
6. 发送 QUIT
```

### 拉取流程 (RECV)

```
1. openService("sync:")
2. 发送 RECV + 远程路径长度 + 路径
3. while (收到 DATA):
     写入本地文件
4. 收到 DONE
5. 发送 QUIT
```

### 目录列表流程 (LIST)

```
1. openService("sync:")
2. 发送 LIST + 远程路径长度 + 路径
3. while (收到 DENT):
     解析 mode, size, mtime, 文件名
4. 收到 DONE
5. 发送 QUIT
```

## 四、FramebufferService

```kotlin
class FramebufferService(private val session: AdbSession) {
    // 截取设备屏幕，返回原始像素数据
    suspend fun capture(): FramebufferResult
}

data class FramebufferResult(
    val version: Int,       // 版本号 (2)
    val bpp: Int,           // bits per pixel
    val colorSpace: Int,    // 颜色空间
    val width: Int,         // 宽度
    val height: Int,        // 高度
    val pixels: ByteArray   // 原始像素数据
)
```

### 实现

```
1. openService("framebuffer:")
2. 读取第一个 WRTE: 解析 header (version, bpp, colorSpace, width, height)
3. 读取后续 WRTE: 拼接像素数据
4. 收到 CLSE
```

## 五、PackageService

```kotlin
class PackageService(private val shellService: ShellService) {
    
    // 安装 APK
    suspend fun install(apkPath: String, options: InstallOptions): ShellResult
    
    // 卸载应用
    suspend fun uninstall(packageName: String, keepData: Boolean = false): ShellResult
}

data class InstallOptions(
    val replace: Boolean = true,          // -r: 替换已有应用
    val grantPermissions: Boolean = false, // -g: 授予所有运行时权限
    val allowDowngrade: Boolean = false,  // -d: 允许降级安装
    val installLocation: String? = null   // -f/-s: 内部/外部存储
)
```

> PackageService 本质上是 ShellService 的封装，通过 shell 执行 `pm install`/`pm uninstall` 命令。未来可扩展为通过 `exec:` 服务直接安装（支持 streamed APK）。

## 六、错误处理策略

```kotlin
sealed class AdbException(message: String, cause: Throwable? = null) : Exception(message, cause) {
    class ConnectionFailed(host: String, port: Int, cause: Throwable) : AdbException(...)
    class AuthenticationFailed(reason: String) : AdbException(...)
    class DeviceNotFound(serial: String) : AdbException(...)
    class ServiceError(service: String, reason: String) : AdbException(...)
    class ProtocolError(message: String) : AdbException(...)
    class StreamClosed(localId: Int) : AdbException(...)
    class CommandFailed(command: String, exitCode: Int, stderr: String) : AdbException(...)
}
```

各服务层方法抛出具体的异常类型，UI 层按需捕获和展示。
