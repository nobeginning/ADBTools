# 传输层设计

## 概述

传输层是整个 ADB Library 的基础，负责 TCP socket 管理、数据包的可靠收发、认证握手，以及并发控制。

## 核心接口

### Transport

```kotlin
// transport/Transport.kt
interface Transport : Closeable {
    suspend fun read(): AdbPacket      // 读取一个完整 packet (header + payload)
    suspend fun write(packet: AdbPacket) // 写入一个完整 packet
    val isConnected: Boolean
}
```

设计原则：
- 最小接口：仅暴露 read/write + 连接状态
- 协程友好：所有方法都是 suspend 函数
- 与协议层解耦：不关心 packet 内容，只负责传输

### 关键实现细节

**1. read() — 分两步读取**

```
步骤1: 读取固定的 24 字节 header
  inputStream.readFully(headerBuffer, 0, 24)
  
步骤2: 解析 header 获取 data_length
  val msg = AdbMessage.parse(headerBuffer)
  
步骤3: 如果 data_length > 0，读取 payload
  val payload = ByteArray(msg.dataLength)
  inputStream.readFully(payload, 0, msg.dataLength)
  
步骤4: 验证 magic（checksum 在现代 ADB 中可跳过）
  check((msg.magic xor msg.command) == 0xFFFFFFFF.toInt()) { "Magic mismatch" }
  // 自 version 0x01000001 起，data_crc32 可为 0，仅非零时做弱校验
```

**2. write() — 原子写入**

```
步骤1: writeMutex.withLock { ... }  保护完整写入
步骤2: 构造 24 字节 header
步骤3: 写入 header (24B)
步骤4: 如果 payload 非空，写入 payload
步骤5: flush()
```

**3. 并发控制**

- **写操作**：Mutex 保护，防止多个协程同时写入导致数据交错
- **读操作**：不需要锁。ADB 协议是请求-响应的，同一时刻只有一个协程在读取
- **读写分离**：读和写可以并发进行（全双工），互不影响

### 超时处理

- `connectTimeout`: TCP 连接超时（默认 5 秒）
- `readTimeout`: socket.setSoTimeout() 设置（默认 30 秒），超时抛出 SocketTimeoutException

## 认证处理

### AuthHandler

```kotlin
// transport/AuthHandler.kt
class AuthHandler(private val keyStoreDir: File) {
    
    // 密钥对，懒加载（首次使用时从文件加载或生成）
    private val keyPair: KeyPair by lazy { loadOrGenerate() }
    
    // 认证主流程
    suspend fun authenticate(transport: Transport): Boolean {
        // 1. 发送 CNXN 连接请求
        transport.write(createConnectPacket())
        
        // 2. 读取响应
        val response = transport.read()
        
        return when (response.command) {
            AdbCommand.CNXN -> true  // 无需认证
            AdbCommand.AUTH -> handleAuthFlow(transport, response)
            else -> throw AdbException.AuthenticationFailed("Unexpected response")
        }
    }
    
    // RSA 签名
    fun signToken(token: ByteArray): ByteArray
}
```

### 密钥持久化

```
文件路径: $keyStoreDir/adb_lib_key  (PEM 格式)

```

## AdbSession — 设备会话

```kotlin
// AdbSession.kt
class AdbSession(
    val device: AdbDevice,
    private val transport: Transport
) : Closeable {
    
    private val localIdCounter = AtomicInteger(1)
    private val streams = ConcurrentHashMap<Int, AdbStream>()
    
    // 打开一个服务流
    suspend fun openService(service: String): AdbStream {
        val localId = localIdCounter.getAndIncrement()
        val packet = AdbPacket(AdbCommand.OPEN, localId, 0, service.toByteArray())
        transport.write(packet)
        
        val response = transport.read()
        when (response.command) {
            AdbCommand.OKAY -> {
                val stream = AdbStream(localId, response.arg1, transport)
                streams[localId] = stream
                return stream
            }
            AdbCommand.CLSE -> throw AdbException.ServiceError(service, "Stream closed")
            else -> throw AdbException.ProtocolError("Expected OKAY or CLSE")
        }
    }
}
```

## AdbStream — 服务流

```kotlin
// AdbStream.kt
class AdbStream(
    val localId: Int,
    val remoteId: Int,
    private val transport: Transport
) : Closeable {
    
    // 写数据到流
    suspend fun write(data: ByteArray) {
        val packet = AdbPacket(AdbCommand.WRTE, localId, remoteId, data)
        transport.write(packet)
    }
    
    // 读取流的下一个 packet（可能是 WRTE 或 CLSE）
    suspend fun read(): AdbPacket {
        val packet = transport.read()
        when (packet.command) {
            AdbCommand.WRTE -> return packet
            AdbCommand.CLSE -> throw AdbException.StreamClosed(localId)
            else -> throw AdbException.ProtocolError("Unexpected packet in stream")
        }
    }
    
    // 关闭流
    suspend fun closeStream() {
        val packet = AdbPacket(AdbCommand.CLSE, localId, remoteId, ByteArray(0))
        transport.write(packet)
    }
}
```

### Stream 生命周期

```
OPEN ────▶ OKAY ────▶ WRTE / WRTE / ... ────▶ CLSE
                      ▲  ▲
                      │  │ (全双工读写)
                      │  │
```

Stream 是全双工的：双方可以同时发送 WRTE 包。local_id 和 remote_id 用于将 WRTE 包路由到正确的流。
